# Security configuration

The application validates configuration at startup and refuses to start when
required values are missing or weak. Copy `.env.example` to `.env`, then
replace every placeholder locally. Never commit the resulting `.env` file.

## Secret requirements

- Database passwords must contain at least 12 characters.
- JWT access and refresh secrets must each contain at least 32 random
  characters and must be different values.
- Production secrets should be generated by a password manager or secret
  manager, not manually chosen.
- Use separate values for development, test, staging, and production.

The previous `.env` file was tracked in Git. Removing it from the current
index does not erase it from existing commits. Rotate the database password,
JWT secret, and refresh secret before deployment. If this repository was ever
shared, use an approved history-cleaning process and coordinate the required
force-push with every collaborator.

## Logging

Request logs contain only request ID, method, path, response status, and
duration. Authorization headers, cookies, request bodies, password hashes,
and environment values must never be logged.

## Repository hygiene

`.env` and every nested `node_modules` directory are ignored. Dependencies are
restored from `package-lock.json` with `npm ci`.
